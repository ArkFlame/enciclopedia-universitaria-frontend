Guía paso a paso para contribuir a la Enciclopedia Universitaria.

Cómo crear un artículo completo y aprobado

Desde redactar el contenido inicial hasta lograr la aprobación: todo lo que necesitás saber.

Antes de empezar

Cada artículo es un aporte académico. Antes de crear uno, asegurate de tener:

  • Un tema bien definido y referencias bibliográficas.
  • Resumen (1–3 oraciones) que explique la intención.
  • Imágenes propias o libres de uso para acompañar el texto.

Necesitás una cuenta activa. Si todavía no tenés una, registrate aquí. Una vez logueado, podés acceder al editor desde el botón Nuevo artículo.

1. Metadatos clave

Completa título (máx. 500 caracteres), resumen descriptivo (máx. 2000) y categoría. Si lo deseás, podés dejar la categoría vacía y asignarla más tarde con colaboración.

  • Título claro y específico.
  • Resumen que resuma el núcleo del artículo.
  • Categoría relevante para que el buscador lo indexe.
2. Redacción en Markdown

Usa el editor principal para escribir en Markdown. Podés insertar títulos, listas, citas y código. La vista previa te ayuda a verificar el formato antes de enviar.

Insertá texto, párrafos y bloques con saltos de línea; los atajos de negrita/cursiva están en la barra superior.

3. Shortcodes interactivos

Aprovechá los componentes disponibles para lograr interactividad segura. Algunos ejemplos:

  • [tooltip text="definición"]palabra[/tooltip]
  • [accordion title="Pregunta"]Respuesta aquí[/accordion]
  • [tabs]...[/tabs] para secciones con pestañas.
  • [interactive-diagram]... para diagramas clickeables.
  • [ref article="slug"]Nombre del artículo[/ref] para enlazar internamente.

Todos los shortcodes se procesan en el backend y se limpian con sanitización antes de almacenarse.

Carga de multimedia

Subí imágenes desde el panel lateral. Cada archivo debe ser menor a 5 MB y se almacena en una carpeta única vinculada al artículo. Luego podés insertarlas en tu contenido con: 

[image src="https://tuservidor/assets/images/{articleId}/foto.jpg" alt="Descripción" caption="Pie opcional"]

Se comprimen en el backend y se generan URLs públicas con acceso controlado.

Controles de calidad

- El contenido pasa a una cola con estado PENDIENTE. Sólo moderadores o administradores pueden aprobar o rechazar. - Si el artículo es rechazado, recibirás el motivo junto al enlace para actualizarlo. - Podés trabajar en nuevas versiones: cada envío incrementa el número de versión.

Publicar y hacer seguimiento

  1. Revisá ortografía y referencias.
  2. Incluí shortcodes necesarios para interactividad (tooltip, callouts, diagramas).
  3. Agregá imágenes subidas usando el gestor multimedia.
  4. Presioná Enviar para revisión. El sistema guarda el artículo como pending.
  5. Recibirás notificaciones en la campana del navbar cuando cambie el estado.
  6. Si querés volver a editar antes de la aprobación, podés cargar el artículo y enviar una nueva propuesta.

Consejos de estilo y seguridad

  • Evita contenido duplicado o sin fuente.
  • No uses HTML arbitrario: el sistema sólo permite shortcodes definidos.
  • Si querés enlaces internos, usá [ref article="slug"]Texto[/ref].
  • Para diagramas interactivos, cada célula puede enlazar un artículo diferente y mostrar más información al hacer clic.
  • Revisá la guía completa de shortcodes desde el botón Guía de shortcodes.

¿Qué sucede después de publicar?

Una vez aprobado, el artículo se publica públicamente. Podés crear contenido adicional, agregar más multimedia y seguir haciendo mejoras. Si llegás al límite de lecturas gratis, el paywall te mostrará un CTA para suscribirte a $1.000 ARS/mes y desbloquear acceso ilimitado.

Recordá que los usuarios FREE tienen límites de edición y publicación (1 artículo/edición por hora). Los usuarios MONTHLY tienen acceso prioritario y sin límites.
